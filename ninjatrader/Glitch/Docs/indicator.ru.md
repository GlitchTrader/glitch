# Индикатор GlitchAnalyticsBridge

## Назначение

`GlitchAnalyticsBridge` публикует рыночный контекст с графика. Он читает NinjaTrader, формирует мультитаймфрейм-данные, может окрашивать бары и публикует в AddOn. Он не выбирает счета и не отправляет ордера.

## Идентичность и значения по умолчанию

- Тип: `NinjaTrader.NinjaScript.Indicators.GlitchAnalyticsBridge`
- Overlay: включён
- Расчёт: `OnPriceChange`
- Публикация при неактивной вкладке: включена
- Таймфреймы: 1, 5, 15 и 60 минут

## Публичные параметры

| Параметр | Назначение |
|---|---|
| `NeutralBand` | Ширина нейтральной области |
| `EnableBarColoring` | Окрашивание баров |
| `PublishToGlitchUi` | Публикация в AddOn |
| `PublishIntervalMs` | Предпочтительный интервал |
| `IntraBarColoring` | Обновление до закрытия бара |
| `PredictiveBoost` | Чувствительность к новому контексту |
| `FlipHysteresis` | Снижение быстрых переключений |
| `PerformanceMode` | Более лёгкий runtime |
| `EnableOrderFlowLayer` | Дополнительный order flow |
| `OrderFlowBlend` | Его вклад |

## Публикуемый контекст

Данные содержат инструмент, таймфрейм, UTC, цену/волатильность, направление/режим, индикаторы, сессию и доступный order flow. Проприетарные формулы не публикуются.

## Публикация и восстановление

Индикатор регистрирует корень и нативный контракт и отвечает на bootstrap-запрос позднее открытого AddOn. Без AddOn расчёт продолжается, а публикация возобновляется после его появления. Устаревшие сохранённые данные не входят в live composite.

## Использование

Добавьте индикатор на нужный график и поддерживайте соединение и новые бары. Закрытый рынок, праздник, обслуживание или разрыв соединения не создают свежих данных.
